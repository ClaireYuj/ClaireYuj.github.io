机器学习基础

L1/L2正则化对比

特性 L1正则化 L2正则化
解的稀疏性 产生稀疏解（特征选择） 非稀疏解
优化形式 $|w|_1$ $|w|_2^2$
梯度特性 不连续梯度（需特殊优化方法） 平滑梯度
适用场景 高维稀疏数据（如文本分类） 通用场景

归一化 vs 标准化

对比维度 归一化（Normalization） 标准化（Standardization）
公式 $(x - x_{\min}) / (x_{\max} - x_{\min})$ $(x - \mu) / \sigma$
分布 强行压缩到[0,1]区间 保留原始分布形态
适用场景 图像处理、固定范围特征 数据分布未知或存在异常值
缺点 对异常值敏感 不保证有界性

KNN算法与k值影响

算法思想：

  1. 计算测试样本与所有训练样本的距离
  2. 选取k个最近邻样本
  3. 通过多数投票（分类）或均值（回归）预测

k值过大问题：

  • 模型偏向全局分布，丧失局部特征
  • 计算开销增大（需存储全部训练数据）
  • 对噪声和无关特征更敏感

5. AUC是什么？

定义：

  • ROC曲线下面积，取值范围[0,1]
  • 物理意义：随机选取正样本得分高于负样本的概率

快速计算：
$$\text{AUC} = \frac{\sum_{i=1}^{m} \sum_{j=1}^{n} I(p_i > p_j)}{m \cdot n}$$
其中$m,n$为正负样本数，$I$为指示函数

AUC推广到非二分类问题

多分类推广：

  • One-vs-Rest：对每个类别计算AUC后取平均
  • Weighted AUC：按类别样本比例加权
  • Ranking Metrics：直接使用NDCG/MAP等排序指标

快速计算优化：

  • 近似采样：随机抽取部分样本对
  • 分桶排序：对预测值分桶后计算近似排名

熵与交叉熵

熵：
$$H(p) = -\sum_{i=1}^n p_i \log p_i$$

  • 描述系统的不确定性，熵越大越混乱

交叉熵：
$$H(p,q) = -\sum_{i=1}^n p_i \log q_i$$

  • 衡量两个概率分布的差异
  • 在机器学习中常用作损失函数（当p为真实分布，q为预测分布时）

优化器

优化器通过调节模型参数，最小化损失函数。

三种梯度下降法对比

类型 优点 缺点
批量梯度下降 稳定收敛，方向准确 内存要求高，计算慢
随机梯度下降 内存小，适合在线学习 震荡大，收敛不稳定
小批量梯度下降 平衡速度与稳定性（常用） 需调批量大小

优化器改进路线

  1. Momentum：引入动量项加速收敛，减少震荡。
  2. Adagrad：自适应学习率，适合稀疏数据。
  3. RMSprop：修正 Adagrad 的学习率衰减问题。
  4. Adam：结合 Momentum 和 RMSprop，自适应学习率 + 动量。
  5. Nadam：Adam + Nesterov 加速梯度。

SGD(随机梯度下降)

$$ \theta_{t+1} = \theta_t - lr * grad(\theta)$$
梯度也就是损失函数loss($\theta$)对$\theta$的偏导，也据说loss function上升或者下降的方向。

def sgd(params, grads, lr=0.01):
    for param, grad in zip(params, grads):
        param -= lr * grad
    return params

Adam(自适应矩估计)

分为动量计算、偏差修正、参数更新三个步骤。

  • 动量计算：
    以动量为一阶矩，自适应学习率为二阶矩。
    一阶矩衰减率$\beta_1 = 0.9$, 二阶矩衰减率$\beta_2 = 0.99$
    $m_t = \beta_1m_{t-1}+ (1-\beta_1) * grad(\theta) $
    $v_t = \beta_2    v_{t-1}+ (1-\beta_2) * grad^2(\theta) $

  • 偏差修正：
    针对冷启动问题，设置初始时刻得零偏置
    $$\hat{m_t} = \frac{m_t}{1-\beta_t^1}, \hat{v_t} = \frac{v_t}{1-\beta_t^2}$$

  • 参数更新
    $$\theta_{t+1} = \theta_{t}-lr * \frac{\hat{m_t}}{\sqrt{\hat{v_t}}+\epsilon}$$
    $\epsilon$=1e-8，防止除零

class Adam:
    def __init__(self, params, lr=0.001, beta1=0.9, beta2=0.999, eps=1e-8):
        self.params = params
        self.lr = lr
        self.beta1 = beta1
        self.beta2 = beta2
        self.eps = eps
        self.m = [np.zeros_like(p) for p in params]  # 一阶矩
        self.v = [np.zeros_like(p) for p in params]  # 二阶矩
        self.t = 0  # 时间步

    def step(self, grads):
        self.t += 1
        for i, (param, grad) in enumerate(zip(self.params, grads)):
            self.m[i] = self.beta1 * self.m[i] + (1 - self.beta1) * grad
            self.v[i] = self.beta2 * self.v[i] + (1 - self.beta2) * (grad ** 2)
            # 偏差校正
            m_hat = self.m[i] / (1 - self.beta1 ** self.t)
            v_hat = self.v[i] / (1 - self.beta2 ** self.t)
            # 更新参数
            param -= self.lr * m_hat / (np.sqrt(v_hat) + self.eps)
        return self.params

协同过滤

Goal：将用户-物品评分矩阵 R 分解为两个低维矩阵 U（用户特征）和 V（物品特征），使得 $R≈U⋅V^T$

协同过滤的概念是什么？

核心思想：

  • 基于用户-物品交互行为（显式/隐式）的相似性进行推荐
  • 两类方法：
    • User-CF：找相似用户，推荐他们喜欢的物品
    • Item-CF：找相似物品，推荐给用户

关键公式（用户相似度计算）：
$$\text{sim}(u,v) = \frac{\sum_{i \in I_{uv}}(r_{ui} - \bar{r}u)(r{vi} - \bar{r}v)}{\sqrt{\sum{i \in I_{uv}}(r_{ui} - \bar{r}u)^2} \sqrt{\sum{i \in I_{uv}}(r_{vi} - \bar{r}_v)^2}}$$

损失函数（带正则化）

[
L = \sum_{(u,i) \in \text{已知评分}} \left( R_{ui} - U_u \cdot V_i^T \right)^2 + \lambda \left( |U|_F^2 + |V|_F^2 \right)
]

  • ( U_u ): 用户 ( u ) 的潜在特征向量
  • ( V_i ): 物品 ( i ) 的潜在特征向量
  • ( \lambda ): 正则化系数

梯度计算

  • 用户特征梯度：
    [
    \frac{\partial L}{\partial U_u} = -2 \left( R_{ui} - U_u \cdot V_i^T \right) V_i + 2\lambda U_u
    ]
  • 物品特征梯度：
    [
    \frac{\partial L}{\partial V_i} = -2 \left( R_{ui} - U_u \cdot V_i^T \right) U_u + 2\lambda V_i
    ]

实现步骤

  1. 数据准备：加载用户-物品评分数据（稀疏格式）。
  2. 参数初始化：随机初始化 ( U ) 和 ( V )。
  3. 迭代优化：
    • 遍历每个已知评分 ( (u, i, R_{ui}) )。
    • 计算预测误差 ( e_{ui} = R_{ui} - U_u \cdot V_i^T )。
    • 更新用户和物品特征：
      [
      U_u \leftarrow U_u + \alpha \left( e_{ui} \cdot V_i - \lambda U_u \right)
      ]
      [
      V_i \leftarrow V_i + \alpha \left( e_{ui} \cdot U_u - \lambda V_i \right)
      ]
  4. 重复：直到损失收敛或达到最大迭代次数。

代码实现

import numpy as np

class MatrixFactorization:
    def __init__(self, num_users, num_items, k=10, alpha=0.01, lambda_reg=0.01, epochs=100):
        self.num_users = num_users
        self.num_items = num_items
        self.k = k          # 潜在因子数量
        self.alpha = alpha  # 学习率
        self.lambda_reg = lambda_reg  # 正则化系数
        self.epochs = epochs

        # 初始化用户和物品特征矩阵
        self.U = np.random.normal(scale=1./self.k, size=(num_users, self.k))
        self.V = np.random.normal(scale=1./self.k, size=(num_items, self.k))

    def fit(self, ratings):
        """训练模型
        ratings: 三元组列表 [(user_id, item_id, rating)]
        """
        for epoch in range(self.epochs):
            total_loss = 0
            for u, i, r in ratings:
                # 计算预测值和误差
                prediction = np.dot(self.U[u], self.V[i])
                error = r - prediction

                # 计算梯度并更新参数
                grad_u = error * self.V[i] - self.lambda_reg * self.U[u]
                grad_v = error * self.U[u] - self.lambda_reg * self.V[i]

                self.U[u] += self.alpha * grad_u
                self.V[i] += self.alpha * grad_v

                # 累计损失（可选）
                total_loss += error**2 + self.lambda_reg * (np.sum(self.U[u]**2) + np.sum(self.V[i]**2))

            print(f"Epoch {epoch+1}, Loss: {total_loss / len(ratings):.4f}")

    def predict(self, u, i):
        """预测用户u对物品i的评分"""
        return np.dot(self.U[u], self.V[i])

Transformer

注意力机制
手写Multi-head Attention公式

计算步骤：

  1. 输入序列$X \in \mathbb{R}^{n \times d}$，生成Q/K/V矩阵：
    $$Q = XW^Q, \ K = XW^K, \ V = XW^V \quad (W^Q, W^K, W^V \in \mathbb{R}^{d \times d_k})$$
  2. 分割为$h$个头：
    $$Q_i = Q[:, i \cdot d_k/h : (i+1) \cdot d_k/h] \quad (\text{同理} K_i, V_i)$$
  3. 单头注意力：
    $$\text{head}_i = \text{Softmax}\left(\frac{Q_iK_i^T}{\sqrt{d_k/h}}\right)V_i$$
  4. 多头拼接与线性变换：
    $$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, …, \text{head}_h)W^O$$
4. Attention的复杂度是多少？

公式：$$O(n^2 \cdot d)$$
解析：

  • 核心计算来自QK^T矩阵乘法（$n \times d$与$d \times n$）
  • 复杂度分解：
    • QK^T计算：$O(n^2 d)$
    • Softmax与加权求和：$O(n^2)$
  • 总主导项为$O(n^2 d)$

MAP（最大后验概率）和似然函数有什么关系？

技术解析：
MAP（最大后验估计）是贝叶斯框架下的参数估计方法，满足：
$$\theta_{MAP} = \arg\max_\theta P(\theta|X) = \arg\max_\theta \underbrace{P(X|\theta)}{\text{似然}} \cdot \underbrace{P(\theta)}{\text{先验}}$$
关系：

  • MAP = 极大似然估计（MLE） × 先验概率
  • 当先验为均匀分布时，MAP退化为MLE

什么情况下，MAP的损失函数可以用NMSE来计算？（高斯噪声）

核心条件：
当数据噪声服从高斯分布且参数先验也为高斯分布时。
推导：

  • 高斯噪声下，似然函数为：
    $$P(X|\theta) \propto \exp\left(-\frac{1}{2\sigma^2}|y - f_\theta(x)|^2\right)$$
  • 高斯先验：
    $$P(\theta) \propto \exp\left(-\frac{1}{2\tau^2}|\theta|^2\right)$$
  • 对数后验：
    $$-\frac{1}{2\sigma^2}\text{MSE} - \frac{1}{2\tau^2}|\theta|^2 + \text{const}$$
    此时优化目标等价于正则化MSE（NMSE）

GBDT vs Bagging

核心区别：

GBDT Bagging
基学习器关系 串行训练，依赖残差 并行训练，相互独立
样本权重 动态调整（错误样本权重增加） 均匀采样（Bootstrap）
方差-偏差平衡 主要降低偏差 主要降低方差
典型算法 XGBoost, LightGBM Random Forest

XGBoost参数调整思路

调参优先级：

  1. 学习率（learning_rate）：0.05~0.3
  2. 树深（max_depth）：3~8
  3. 正则化（lambda, alpha）：0.1~10
  4. 采样率（subsample, colsample_bytree）：0.6~1.0
